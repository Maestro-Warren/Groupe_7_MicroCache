Utilisation de MicroCache

Explications détaillées sur le protocole, la connexion TCP, les commandes et les scénarios d’usage typiques.

Protocole MicroCache

MicroCache utilise un protocole texte simple basé sur des lignes terminées par `\n`. Chaque commande est traitée séquentiellement et renvoie une réponse textuelle.

PING                    → PONG
SET key value           → OK
SET key value EX 30     → OK
GET key                 → value ou NIL
EXISTS key              → 1 ou 0
DEL key                 → OK

Exemple de client TCP

Voici un exemple en Node.js qui illustre la connexion au serveur MicroCache et l’envoi de commandes.

const net = require('net');

const client = net.createConnection({ port: 6379 }, () => {
  client.write('SET concert:1:tickets 150\n');
  client.write('GET concert:1:tickets\n');
});

client.on('data', (data) => {
  console.log(data.toString());
});

Scénarios d'usage

  • Requêtes de lecture : prioriser le cache pour obtenir une réponse rapide.
  • Mise à jour de données : invalider ou mettre à jour le cache après modification.
  • Monitoring : évaluer le hit rate et la latence pour optimiser le cache.

Modes de lecture

Le projet compare deux chemins :

  • Avec cache : l’API interroge MicroCache, reçoit la valeur si elle est présente.
  • Sans cache : l’API lit directement depuis SQLite pour comparaison de performance.
