Tests de performance

Validation et benchmarks

Guide des tests de charge et interprétation des résultats pour mesurer les gains de performance apportés par MicroCache.

Script de test de charge

Le script `loadtest.js` simule des requêtes concurrentes afin de comparer la performance en mode cache et sans cache.

node src/loadtest.js
node src/loadtest.js --requests 500 --concurrency 50 --concertId 4 --duration 30

Options du test

  • --requests : nombre total de requêtes générées.
  • --concurrency : niveau de requêtes simultanées.
  • --concertId : identifiant du concert ciblé.
  • --duration : durée totale du test en secondes.

Analyse des résultats

  • Rappel : la latence est le principal bénéfice du cache pour les lectures répétées.
  • Hit rate élevé : montre que les requêtes sont souvent servies par le cache.
  • Faible latence : améliore l’expérience utilisateur sous forte charge.
