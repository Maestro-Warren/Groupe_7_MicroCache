Documentation API

Endpoints de l'API MicroCache

Liste complète des routes, des paramètres, des réponses et du comportement de l’API de billetterie.

Endpoints principaux

GET

/api/concerts

Retourne la liste des concerts disponibles avec leur nombre de places restants et leurs métadonnées.

Réponse : JSON listant les concerts.
GET

/api/tickets?concertId={id}

Récupère le nombre de places restantes pour un concert via MicroCache.

Réponse : nombre de places et statut cache.
GET

/api/tickets/nocache?concertId={id}

Interroge directement la base de données sans passer par le cache pour les comparaisons de performance.

Réponse : valeur issue de SQLite.
GET

/api/stats

Retourne les métriques de performance du système : hit rate, latence, requêtes totales, comparateur cache/non-cache.

Réponse : JSON d’indicateurs.

Paramètres et usage

Utilisez `concertId` dans les requêtes de tickets pour cibler un événement précis. Les routes sont conçues pour être simples à tester via curl ou Postman.

curl http://localhost:3000/api/tickets?concertId=1
curl http://localhost:3000/api/tickets/nocache?concertId=1
curl http://localhost:3000/api/stats

Comportement des routes

  • /api/concerts renvoie la liste complète de concerts et leur disponibilité.
  • /api/tickets interroge MicroCache puis SQLite en fallback si nécessaire.
  • /api/tickets/nocache ignore le cache pour mesurer la latence brute.
  • /api/stats expose les métriques du dashboard et des tests.
