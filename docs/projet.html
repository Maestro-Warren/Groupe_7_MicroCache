Objectif du projet

MicroCache est conçu pour démontrer l’efficacité d’un cache distribué dans un système de billetterie. Il met en évidence la différence de performance entre des requêtes directes sur la base de données et des requêtes servies par un cache en mémoire.

Ce projet combine :

  • Un serveur TCP performant en Rust
  • Une API Express en Node.js
  • Un dashboard animé en temps réel
  • Un protocole texte simple pour le cache
  • Un benchmark de charge mesurant les gains

Principaux bénéfices

Latence réduite

La latence moyenne chute de 87ms à 4ms lorsque le cache sert les requêtes.

Évolutivité

Le modèle async Tokio et le store thread-safe permettent une montée en charge naturelle.

Visibilité

Le dashboard fournit des données de hit rate, de requêtes/sec et de latence en direct.

Durabilité

La persistence via snapshot binaire assure une restauration rapide du cache.

Architecture technique

Schéma des composants et du flux de données entre API, cache et base de données.

Module Rust

  • server.rs : serveur TCP avec Tokio.
  • store.rs : cache thread-safe + TTL.
  • parser.rs : analyse du protocole texte.
  • reaper.rs : nettoyage des clés expirées.
  • persistence.rs : snapshot binaire du store.

API Node.js

  • index.js : serveur Express et WebSocket.
  • cacheClient.js : client TCP vers MicroCache.
  • db.js : accès SQLite et données de concert.
  • stats.js : collecte des métriques.
  • routes/tickets.js : endpoints de billetterie.

Dashboard & Tests

  • Interface Dashboard avec graphiques et statut WebSocket.
  • Script de charge `loadtest.js` pour comparer performance cache/non-cache.
  • Visualisation de latence, hit rate, req/sec, et journal des requêtes.

Protocole MicroCache

Commandes prises en charge

PING → PONG
SET key value → OK
SET key value EX 30 → OK
GET key → value ou NIL
EXISTS key → 1 ou 0
DEL key → OK

Comportement

Le protocole reste simple et lisible. Les requêtes sont terminées par un saut de ligne et chaque réponse est envoyée en texte clair. Cela facilite l’intégration avec des clients TCP légers et des tests manuels.

Lancement du projet

Démarrage Rust

Compiler et lancer le serveur MicroCache :

cd Groupe_7_MicroCache
cargo run

Démarrage API

Installer et lancer l’API demo :

cd microcache-demo/api-server
npm install
npm start

Test de charge

Comparer le comportement via le script de test :

node src/loadtest.js
node src/loadtest.js --requests 500 --concurrency 50 --concertId 4 --duration 30
